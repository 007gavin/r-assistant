1. 安装

从本地 tar.gz 安装

# 指定包文件路径，不从CRAN下载
install.packages("D:/r-assistant/r.assistant_0.1.0.tar.gz",
                 repos = NULL, type = "source")

从源码目录安装

# 如果包还没有打包成 tar.gz
install.packages("D:/r-assistant", repos = NULL, type = "source")

依赖包

自动安装

安装时 R 会自动安装以下依赖：httr2、jsonlite、rstudioapi、miniUI、shiny、clipr

2. 配置 API

首次使用前需要设置 LLM 提供商和 API Key。

方式一：命令行配置（推荐）

library(r.assistant)

# 设置提供商和 API Key（一步到位）
assistant_config(
  provider  = "deepseek",
  api_key   = "sk-xxxxxxxxxxxxxxxx"
)

# 或者分开设置
assistant_set_provider("deepseek")
assistant_set_key("sk-xxxxxxxxxxxxxxxx")
assistant_set_model("deepseek-chat")

方式二：图形界面配置（仅 RStudio）

# 不带参数调用，弹出配置面板
assistant_config()
API Key 存储位置

API Key 保存在本地文件 ~/.r-assistant/config.json，不会上传到任何地方。

完整配置选项

assistant_config(
  provider         = "deepseek",     # LLM 提供商
  model            = "deepseek-chat", # 模型名称
  api_key          = "sk-xxx",        # API 密钥
  base_url         = "",               # 自定义 API 地址
  temperature      = 0.3,              # 温度 (0-2)，越低越确定
  max_tokens       = 4096,             # 最大回复长度
  system_prompt    = "...",           # 自定义系统提示词
  context_enabled  = TRUE             # 是否包含 R 会话上下文
)

# 查看当前配置
assistant_get_config()

3. 支持的 LLM 提供商

提供商 默认模型 环境变量 说明
deepseek deepseek-chat DEEPSEEK_API_KEY 国内直连，推荐
openai gpt-4o-mini OPENAI_API_KEY ChatGPT 系列
anthropic claude-sonnet-4 ANTHROPIC_API_KEY Claude 系列
openrouter deepseek/deepseek-chat OPENROUTER_API_KEY 一个 Key 访问多种模型
custom 自定义 R_ASSISTANT_API_KEY 任何 OpenAI 兼容 API 
# 查看所有可用提供商
assistant_available_providers()
# [1] "openai" "anthropic" "deepseek" "openrouter" "custom"

# 使用自定义 API（例如本地 Ollama）
assistant_config(
  provider = "custom",
  base_url = "http://localhost:11434/v1",
  model    = "qwen2.5:14b",
  api_key  = "ollama"
)

4. 对话问答

多轮对话（保留上下文）

assistant_chat("如何用 ggplot2 画一个带有置信区间的折线图？")
# --- R Assistant ---
# 你可以使用 geom_ribbon() 来添加置信区间...
# ```r
# ggplot(data, aes(x, y)) +
#   geom_ribbon(aes(ymin = lower, ymax = upper), alpha = 0.2) +
#   geom_line()
# ```
# -------------------

# 继续追问（自动保留上文）
assistant_chat("再加上颜色分组")
# AI 知道你在说上面的折线图，直接给出修改后的代码

快速提问（一次性，不保留历史）

assistant_ask("lapply 和 sapply 的区别是什么？")
# 直接返回回答，不影响对话历史
提示

assistant_chat 会自动将当前 R 会话的上下文（加载的包、变量等）发送给 AI，因此 AI 的回答会更贴合你的实际环境。

5. 代码解释

# 直接传入代码字符串
assistant_explain("mtcars %>% group_by(cyl) %>% summarise_all(mean)")

# 或在 RStudio 中选中代码后运行（自动读取选中内容）
assistant_explain()

# 控制详细程度
assistant_explain(code, detail_level = "brief")    # 2-3 句简述
assistant_explain(code, detail_level = "normal")   # 默认
assistant_explain(code, detail_level = "detailed") # 逐步详解 + 边界情况

6. 代码重构

# 将 base R 代码重构为 tidyverse 风格
assistant_refactor(
  "for(i in 1:nrow(df)) { df$x[i] <- df$y[i] * 2 }",
  style = "tidyverse",
  goal  = "readability"
)
# 返回重构后的代码 + 修改说明
# ```r
# df <- df %>% mutate(x = y * 2)
# ```

# 参数选项
# style: "tidyverse" | "base" | "data.table"
# goal:  "readability" | "performance" | "conciseness"

7. 错误修复

# 方式一：在 RStudio 中选中报错的代码，然后运行
assistant_fix()
# 自动读取：选中的代码 + 最后一条错误信息

# 方式二：手动指定代码和错误信息
assistant_fix(
  code  = "df %>% filter(x = 5)",
  error = "Error in filter(): could not find function '%>%'"
)
# 解释：缺少 dplyr 包的加载
# 修复：library(dplyr)  或  df[df$x == 5, ]

8. 代码与文档生成

代码生成

用自然语言描述需求，AI 生成 R 代码

文档生成

为函数自动生成 roxygen2 格式文档

测试生成

为函数自动生成 testthat 单元测试

代码插入

生成的代码可一键插入编辑器或复制到剪贴板

# 从描述生成代码
assistant_complete("创建一个函数，输入数据框和列名，返回正态性检验结果")

# 为函数生成 roxygen2 文档
assistant_document("my_summarise <- function(df, grp, val) {
  df %>% group_by(across(all_of(grp))) %>% summarise(mean = mean(.data[[val]]))
}")

# 为函数生成单元测试
assistant_test("add <- function(a, b) a + b")

9. RStudio Addin

安装后，在 RStudio 菜单栏 Addins 下拉菜单中可看到以下工具：

Addin 名称 对应函数 功能
R Assistant Chat addin_chat() 打开聊天面板，支持多轮对话
R Assistant: Explain Code addin_explain() 解释选中的代码
R Assistant: Refactor Code addin_refactor() 重构选中的代码
R Assistant: Fix Code addin_fix() 修复选中代码的错误
R Assistant: Generate Docs addin_document() 为选中函数生成文档
快捷键设置

推荐在 Tools → Modify Keyboard Shortcuts 中搜索 "R Assistant" 设置快捷键，例如：

R Assistant Chat → Ctrl+Shift+A

代码插入选项

当 AI 返回包含代码块的回复时，使用 Addin 会弹出选项：

  1. Replace selected code — 用 AI 生成的代码替换编辑器中选中的代码
  2. Insert after selection — 插入到选中代码的后面
  3. Copy to clipboard — 复制到剪贴板
  4. Do nothing — 仅显示，不做任何操作

10. 上下文感知

R Assistant 会自动采集当前 R 会话的信息发送给 AI，无需手动提供：

上下文类型 采集内容 控制参数
会话信息 R 版本、操作系统、工作目录 始终包含
已加载包 当前加载的 R 包列表（排除 base 包） include_session
全局变量 变量名、类型、维度、大小（最多30个） include_env
当前文档 RStudio 打开的文件内容（截断 3000 字符） include_document
选中代码 RStudio 编辑器中选中的文本 include_selection 
# 查看当前采集到的上下文
ctx <- assistant_context()
str(ctx)

# 关闭上下文（减少 token 消耗）
assistant_config(context_enabled = FALSE)

# 单次调用关闭上下文
assistant_chat("什么是管道符？", use_context = FALSE)

11. 对话历史

# 查看最近 N 条对话
assistant_history(n = 5)

# 清空所有历史
assistant_clear_history()
存储位置

对话历史保存在 ~/.r-assistant/history.json，默认最多保留 50 条。可在配置中调整：

# 修改历史保留条数（在配置文件中）
assistant_config()
# 修改 history_max 字段即可
